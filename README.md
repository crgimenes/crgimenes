I've been programming since I was 13. Most of that time went into systems nobody sees -- internal tools, infrastructure, the kind of code that doesn't show up in a portfolio but keeps things running.

These days I work mostly with Go, C, UNIX and SQLite. I build tools I use, not frameworks that show up in conference talks. I prefer code that lasts and systems you can fully understand -- not out of principle, but because code nobody can read eventually becomes someone else's problem.

## What I build

A few projects worth pointing at:

- **[neko](https://github.com/crgimenes/neko)** -- the classic cat that chases your mouse pointer, rewritten in Go with Ebitengine. Mostly nostalgia. Runs on macOS, Linux and Windows.
- **[glaze](https://github.com/crgimenes/glaze)** -- a WebView toolkit for desktop apps in Go. CGO-free where possible, because dragging a C toolchain into a Go project just to render HTML felt wrong.
- **[filo](https://github.com/crgimenes/filo)** -- a small scripting language meant to be embedded in Go programs. I tried the obvious options first. None of them fit.
- **[compterm](https://github.com/crgimenes/compterm)** -- terminal sharing over the network. Started as a teaching aid for the study group, then turned out to be useful elsewhere.
- **[dbv](https://github.com/crgimenes/dbv)** -- a PostgreSQL viewer for the terminal. I wanted to read tables without leaving tmux.
- **[shutdown](https://github.com/crgimenes/shutdown)** -- a `shutdown` command for MS-DOS, written in 16-bit C. Pure indulgence.

There are more in the tab above. Most of them are small on purpose.
